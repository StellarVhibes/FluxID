## ⭐️ About the Program

**Stellar Journey to Mastery** is a hands-on builder journey — not a course, not a bootcamp, not a hackathon. It's a structured, level-based progression system that takes developers from basic wallet connections all the way to launching production-ready dApps on Stellar mainnet.

### What You'll Do:
- ⚡ Build real dApps and smart contracts
- 🏗️ Ship production-ready projects
- 👥 Onboard real users
- 🌍 Launch on Mainnet
- 💰 Compete for a **$20,000 monthly prize pool**
- 🚀 Unlock startup opportunities and ecosystem funding

Whether you're a student, hacker, indie builder, or aspiring founder — this program rewards **building in public** and creating things that actually matter.

## 🥋 Program Structure & Tracks

### 1️⃣ Builder Track — Belt System

Progress through 7 belt levels by building real applications on Stellar. Each level has specific technical requirements.

| Level | Belt | Focus |
|-------|------|--------|
| 1 | ⚪️ **White Belt** | Wallets, Contracts, Transactions, Multi-wallet |
| 2 | 🟡 **Yellow Belt** | Multi-wallet, contracts & events, writing contracts |
| 3 | 🟠 **Orange Belt** | Complete mini dApp + tests + Production basic + Advanced contract + **Idea Submission** (unique ideas resonating with Stellar Anchors) |
| 4 | 🟢 **Green Belt** | Build Production MVP + 10 users (testnet) |
| 5 | 🔵 **Blue Belt** | Scale to 50 users + add features based on feedback + Pitch + Demo |
| 6 | ⚫️ **Black Belt** | Mainnet Launch + Audit + Real Adoption + 10+ mainnet users + Demo Day ready |
| 7 | 🏆 **Master Belt** | Startup track with SCF grants and InstaAward (selective projects) |

> **Key Rule:** The belt chain must remain **unbroken**. If you fail Level 3, you only earn rewards up to Level 2 — even if you submitted Level 4+.

## ❓ Key FAQs

**Q: Can I join with a project I've already started?**  
A: Yes, but submissions must meet specific belt requirements and reflect **new progress** made during the program period.

**Q: Can I progress multiple belts in one month?**  
A: Yes! You can progress as fast as you can build. However, you only earn the **prize pool for the highest belt reached**, not every level passed.

**Q: Can I switch projects month to month?**  
A: The program encourages progression on one project. Pivoting is allowed in the following month **only with mentor approval**.

**Q: What happens after Master Belt?**  
A: The Master Belt is ongoing. Continue shipping major features and onboarding mainnet users each month to remain in the reward cycle.

**Q: Once I earn a reward for a belt, can I compete at that same belt again?**  
A: No — once rewarded at a level, you must move up. **Exception:** Master Belt (Level 7) allows monthly re-entry by continuously improving your product.

**Q: Are there live sessions?**  
A: Yes — regular office hours, technical workshops, and live Q&A sessions. Participation is optional. Check community channels for schedules.

## 👀 Judging Criteria

To ensure a fair and high-quality experience, our technical committee selects winners based on a **comparative, holistic review**:

| Criteria | Description |
|----------|-------------|
| ✅ **Core Technical Standards** | Strong functionality, strict adherence to level requirements, and comprehensive documentation |
| 🧹 **Code Quality & Security** | Clean, well-structured, and efficient code. For higher levels, smart contract security and optimization are key evaluation criteria |
| 🌍 **Ecosystem Fit** | Is your project solving a real problem or filling a gap in the Stellar ecosystem? Projects with high utility are prioritized |
| 👥 **User Traction** *(Levels 5, 6 & 7)* | For Blue Belt, Black Belt, and Master Belt — technical completion is only half the battle. You must demonstrate active user onboarding and real-world interaction with your application |

### Belt-by-Belt Evaluation Breakdown

```
Level 1–4  → Technical quality + belt requirements + documentation
Level 5     → Above + 50 testnet users + pitch/demo + mentor approval
Level 6     → Above + mainnet launch + audit + 10+ real mainnet users
Level 7     → Ongoing: new features + growing mainnet user base
```
